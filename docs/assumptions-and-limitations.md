# Assumptions and Limitations

## Reporting assumptions

The prototype reporting workflow assumed that:

- user inputs were entered in the expected structure
- the forecasting component returned reporting-ready outputs
- field names and data types remained consistent between steps
- the Power BI view refreshed from the intended input source
- users understood that scenario outputs depended on the supplied assumptions

## Interpretation limitations

- Forecasting outputs are estimates, not guaranteed outcomes.
- A clear dashboard does not remove uncertainty from the underlying model.
- Results should be interpreted together with assumptions, data coverage, and model limitations.
- Prototype behaviour does not demonstrate production reliability or deployment readiness.
- No business-impact claim can be made without an approved operational evaluation.

## Portfolio limitations

This public repository cannot independently reproduce the original workflow because the source data, internal files, dashboard file, and team code are not published.

The repository instead demonstrates:

- how I describe the reporting problem
- what I personally contributed
- how I think about assumptions and handover
- how I communicate confidential work safely
