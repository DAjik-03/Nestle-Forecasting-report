# Confidentiality and Public Scope

## Purpose of this repository

This repository provides enough information for a recruiter to understand the project context, my contribution, and the analyst skills demonstrated without exposing confidential material.

## Information allowed in this public case study

- the fact that the work was an AUT academic-industry collaboration with Nestlé New Zealand
- high-level descriptions of forecasting-oriented KPI reporting
- general tools such as Power BI, Excel, documentation, and supporting Python workflow
- my own contribution described without internal business detail
- general reporting lessons, assumptions, and limitations

## Information excluded

- product or item names
- retailer, category, pricing, promotional, margin, or cost details
- internal percentages, forecasts, KPIs, targets, or performance figures
- Nestlé-provided data or files
- database files, notebooks, dashboards, spreadsheets, presentations, and handover packages
- client communications and meeting records
- personal information about client or team members
- code or written work owned by other contributors

## Publication rule

If a future portfolio asset cannot be understood without revealing internal context, it should not be published. A public-safe recreation may only be added when it uses synthetic information, is clearly labelled as a recreation, and does not imply that invented results came from the original project.

## Disclaimer

This repository is an independent personal portfolio summary. It is not an official Nestlé repository, publication, endorsement, or production system. The collaboration was academic and is not presented as professional employment.
